ROUTING LOGIC:
  User Query
    ↓
  MainAgent analyzes complexity
    ↓
  Simple? → Direct tool execution (Calculator, WebSearch, etc.)
  Complex? → Delegate to sub-agent(s) (Web, Code, Database, Search)
    ↓
  Combine results
    ↓
  Stream response to user (via WebSocket)

TOOL EXECUTION PATTERN:
  1. LLM sees tool schema
  2. LLM decides which tool to use
  3. Tool.validate(params)
  4. Tool.executeWithRetry() [auto-retry + timeout]
  5. Return result to agent

FULL REQUEST FLOW
================================================================================

User sends message
    ↓
REST API: POST /new
    ↓
aiController.sendMessage()
    ↓
Create Conversation & Message records
    ↓
Get MainAgent from registry
    ↓
MainAgent.execute(context)
    ├─ Load relevant memories (MemoryManager)
    ├─ Analyze query complexity
    ├─ Route decision:
    │   ├─ Simple → _executeWithTools()
    │   │   ├─ Get tool schemas
    │   │   ├─ Send to LLM with tools available
    │   │   ├─ LLM calls tool
    │   │   ├─ Execute tool via ToolRegistry
    │   │   └─ Return result
    │   │
    │   └─ Complex → _delegateToSubAgents()
    │       ├─ Determine which agents needed
    │       ├─ TaskDelegator.delegate()
    │       │   └─ AgentMessenger.send()
    │       │       └─ Sub-agent executes
    │       ├─ Collect results
    │       └─ Synthesize response
    │
    ├─ Stream progress via WebSocket
    ├─ Extract & store new memories (MemoryManager)
    └─ Return final response
    ↓
Response streamed to client
    ↓
Message saved to database
    ↓
User sees response in UI

KEY PATTERNS
================================================================================

1. SINGLETON SERVICES
   Every service is instantiated once and exported
   module.exports = new ServiceClass();
   Usage: const service = require('../services/ServiceName');

2. TOOL REGISTRY PATTERN
   Tools registered at startup, accessed by name
   toolRegistry.register('name', toolInstance);
   toolRegistry.get('name').execute(params);

3. AGENT REGISTRY PATTERN
   Agents registered at startup, routed by MainAgent
   agentRegistry.register('name', agentInstance);
   agentRegistry.get('name').execute(context);

4. SKILL → SERVICE → DATABASE
   Each skill maps to one or more services
   Services implement business logic
   Database tables store persistent data

5. DELEGATION PATTERN
   MainAgent routes to sub-agents via TaskDelegator
   Sub-agents receive structured task objects
   Results aggregated and returned to user

6. SOUL.MD ALIGNMENT
   All agents follow soul.md behavioral principles
   Every skill execution should align with soul.md
   Agents should be curious, empathetic, honest

================================================================================

ADDING NEW CAPABILITY
================================================================================

New Skill:
  1. /skills/new-skill/SKILL.md (definition)
  2. /services/NewService.js (singleton implementation)
  3. /skills/new-skill/scripts/*.js (CLI tools)
  4. Database tables (in SKILL.md)
  5. /routes/new-resource.js (API endpoints)
  6. Register in config/agentInit.js or config/toolInit.js

New Tool:
  1. /tools/shared/MyTool.js (extend Tool base class)
  2. Register in /config/toolInit.js
  3. Immediately available to all agents

New Agent:
  1. /agents/agent-type/MyAgent.js (extend Agent base class)
  2. Register in /config/agentInit.js
  3. MainAgent can delegate tasks to it
